# Skills Catalog

This directory contains the canonical schemas and generated catalog for the Qredence Skills Registry.

## Files

### `schema.skill.json`
**Authoritative JSON Schema** for `skill.yaml`. Every skill's metadata file must validate against this schema.

- Defines required fields, data types, patterns
- Enforces naming conventions (kebab-case for IDs, semver for versions)
- Validates DSPy module/signature import paths
- Specifies permission and safety requirements

### `schema.catalog.json`
JSON Schema for the generated `skills.json` catalog file.

### `skills.json` 
**Generated file - do not edit manually**. This catalog is automatically generated by `tools/validate.py` from all `skill.yaml` files in the repository.

The catalog provides:
- Searchable index of all skills
- Quick lookup by ID, tags, or safety level
- Permission overview
- Version information

## Catalog Generation

The catalog is deterministically generated:

```bash
python tools/validate.py
```

This command:
1. Discovers all `skills/*/skill.yaml` files
2. Validates each against `schema.skill.json`
3. Generates `skills.json` with stable sort order (by ID)
4. Fails if the generated catalog differs from the committed version

## Versioning Rules

Skills follow **Semantic Versioning** (semver):

- **MAJOR** (X.0.0): Breaking changes to inputs, outputs, or behavior
- **MINOR** (0.X.0): Backwards-compatible new features or improvements
- **PATCH** (0.0.X): Backwards-compatible bug fixes

### Breaking Changes

Examples of breaking changes that require a major version bump:

- Removing or renaming input/output fields
- Changing field types incompatibly
- Changing behavior in ways that break existing use cases
- Adding new required fields without defaults
- Changing determinism guarantees

### Non-Breaking Changes

Examples that can be minor or patch versions:

- Adding optional input fields
- Adding new output fields
- Improving accuracy or quality
- Bug fixes that restore documented behavior
- Performance improvements
- Documentation updates

## Tags

Tags are used for discoverability. Use lowercase kebab-case.

### Recommended Tag Categories

**Domain**:
- `nlp`, `vision`, `audio`, `multimodal`
- `web`, `documents`, `code`, `data`

**Task Type**:
- `summarization`, `transformation`, `generation`, `extraction`
- `classification`, `analysis`, `planning`, `reasoning`

**Use Case**:
- `content-creation`, `research`, `automation`, `development`
- `analysis`, `organization`

**Properties**:
- `deterministic`, `streaming`, `batch`, `interactive`
- `low-latency`, `high-accuracy`

## Safety Levels

### Low
- No network access
- No file system access
- Pure computation on provided inputs
- No external dependencies beyond DSPy

### Medium
- Controlled external resources (APIs with user-provided keys)
- Read-only file access to specific directories
- May cache data temporarily
- Clear data retention policies

### High
- Write access to file system
- Executes arbitrary code
- Handles sensitive data (PII, credentials)
- Requires additional security review

## Permissions

All skills must explicitly declare permissions:

```yaml
permissions:
  network: false           # Can make HTTP requests
  filesystem_read: false   # Can read files
  filesystem_write: false  # Can write files
  external_tools: []       # e.g., ["browser", "shell"]
```

**Default is false/empty** - skills must opt-in to permissions.

## Updating the Catalog

The catalog is automatically regenerated in CI. If you add or modify a skill:

1. Update `skill.yaml`
2. Run `python tools/validate.py` locally
3. Commit both the skill changes and the updated `catalog/skills.json`
4. CI will verify the catalog is correct

## Querying the Catalog

The catalog is a simple JSON file that can be queried programmatically:

```python
import json

with open('catalog/skills.json') as f:
    catalog = json.load(f)

# Find skills by tag
nlp_skills = [s for s in catalog['skills'] if 'nlp' in s['tags']]

# Find safe skills (no network access)
safe_skills = [s for s in catalog['skills'] 
               if not s['permissions']['network']]

# Find by safety level
low_risk = [s for s in catalog['skills'] if s['safety_level'] == 'low']
```

Or use the CLI:

```bash
# List all skills
python tools/validate.py --list

# Filter by tag
python tools/validate.py --list --tag nlp

# Show detailed info
python tools/validate.py --info web_summarizer
```
