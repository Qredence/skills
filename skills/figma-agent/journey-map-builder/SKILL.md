# Journey Map Builder

## Capability Mode

`Create when supported; otherwise specify`. Build on the current Figma Design canvas when editing is available. Do not assume FigJam execution.

## Required Context

- Source research or a clearly described experience
- Journey start and end points
- Persona or user segment when multiple groups differ

Proceed with labeled assumptions only when they do not invent research findings.

## Workflow

1. Define the journey scope and 4-7 meaningful stages.
2. Extract actions, thoughts or feelings, touchpoints, pain points, and positive moments for each stage.
3. Trace major claims to supplied evidence; never invent quotes.
4. Derive specific opportunities from the evidence and label hypotheses separately.
5. Structure the map as stages by columns and evidence categories by rows.
6. Add an emotion trend only when the source supports relative highs and lows.
7. Create the artifact in Figma Design when supported and requested; otherwise return a complete layout specification.

## Guardrails

- Do not claim FigJam access unless explicitly available.
- Separate observed evidence, interpretation, and opportunity hypotheses.
- Preserve conflicting research instead of averaging it away.
- Do not present inferred emotions or stages as direct findings.

## Output Contract

Return the stage-by-row journey map, evidence references, top opportunities, assumptions, and whether the map was created or proposed.
